### New Phase 8 Findings — Comparison PDF & COREP/Pillar III

- [ ] **D3.36** — **CRR Art. 126 CRE implemented as binary whole-loan; regulation requires proportion-based split.** `crr_risk_weights.py` `calculate_commercial_re_rw()` and `calculator.py` lines 984–997 apply 50% if LTV ≤ 50% with income cover, 100% otherwise. Art. 126(2)(d) requires: 50% on the part of the loan not exceeding 50% of MV (or 60% of MLV), counterparty RW on the excess — analogous to Art. 125 RRE split which IS correctly implemented as a weighted blend. For LTV ≤ 50% the result is identical (whole loan within secured portion), but for LTV > 50% with qualifying conditions met, the code over-assigns risk. - [ ] **D3.59** — **Art. 124(4) mixed RE split not implementable under current input schema.** Discovered during D2.42 closure (2026-04-22). PRA PS1/26 Art. 124(4) requires a single RE exposure secured by both residential and commercial property to be split proportionally by property value, with each part routed through its own risk-weight article (Art. 124F–I if both parts separately qualify under Art. 124A; Art. 124J on both parts otherwise). The current input schema in `src/rwa_calc/data/schemas.py` exposes a single `property_value` and `property_type` per exposure row, making the proportional split impossible in-engine — firms must currently pre-split mixed-use exposures into two input rows at the loader boundary (documented as a workaround in the new B31 SA spec Art. 124 section). To implement Art. 124(4) natively the schema needs either (a) new optional fields `residential_property_value` / `commercial_property_value` on Loan / Contingent with a dedicated `is_mixed_re` Boolean flag, or (b) a repeated-collateral structure allowing N property rows per exposure. The B31 SA calculator branch in `src/rwa_calc/engine/sa/namespace.py` then needs an `is_mixed_re` fan-out path that computes per-component EAD and applies the Art. 124A gate to each component's qualifying status separately (Art. 124J is the default; Art. 124F–I applies only when both parts qualify). Spec-side now complete at `docs/specifications/basel31/sa-risk-weights.md#real-estate--framework-scope-art-124`. **Effort: M** | Ref: PRA PS1/26 Art. 124(4); `src/rwa_calc/data/schemas.py`; `src/rwa_calc/engine/sa/namespace.py`

- [ ] **D3.60** — **Art. 138(1)(g) implicit-support prohibition and Art. 139(6) higher-of rule not implemented — input-schema gap.** The facility schema exposes a single `external_cqs` (post Art. 138 resolution) with no field distinguishing **issue-specific** from **general issuer** ratings (prerequisite for Art. 139(6)(b)), and no field flagging whether an ECAI assessment **incorporates implicit government support** (prerequisite for Art. 138(1)(g) suppression and the Art. 139(6) clean-vs-with-support comparison).
